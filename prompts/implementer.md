## 작업

기획서의 요구사항을 할당된 접근법에 따라 구현하세요:

1. 위 프로젝트 컨텍스트를 참조하여 **기존 패턴과 컨벤션을 따르세요**
2. **할당된 접근법의 아키텍처 결정을 따르세요**
3. **권장 라이브러리를 사용**하세요
4. **프로덕션 수준의 깨끗한 코드**를 작성하세요
5. **필요한 설정 파일**을 추가/수정하세요

## 구현 지침

- 위 컨텍스트에 핵심 코드가 포함되어 있으므로 **전체 프로젝트 탐색은 불필요**합니다
- 필요한 파일만 선택적으로 참조하고 **바로 구현에 집중하세요**
- 기존 프로젝트의 **코딩 컨벤션**을 따르세요
- 적절한 **관심사 분리**를 유지하세요
- 새 의존성이 필요하면 **의존성 파일**을 업데이트하세요 (build.gradle, package.json 등)

## 중요

- 이 구현은 리뷰와 테스트를 거칩니다
- 프로덕션에 나갈 코드처럼 작성하세요
- TODO/placeholder 주석 없이 완전히 구현하세요
- 할당된 접근법에 충실하세요
- 변경 사항은 이 worktree의 git 브랜치에 자동으로 기록됩니다

## 필수 작업

**반드시 다음을 수행하세요:**

1. **실제 코드 파일 작성** - Write 도구를 사용하여 필요한 모든 파일을 생성/수정하세요
   - 새 파일 생성: Write 도구 사용
   - 기존 파일 수정: Edit 도구 사용
   - 계획만 설명하지 말고, 실제로 파일을 작성하세요!

2. **work-done.md 작성** - 구현 완료 후 반드시 다음 형식으로 작성하세요:
   ```markdown
   # 구현 완료: {approach_name}

   ## 구현 요약
   - 핵심 기능을 어떻게 구현했는지 3-5줄로 설명

   ## 생성/수정된 파일
   - src/main/java/Foo.java: 설명
   - config/application.yml: 설명

   ## 실행 방법
   \`\`\`bash
   ./gradlew bootRun
   \`\`\`

   ## 기술적 결정
   - 왜 이 라이브러리를 선택했는지
   - 중요한 설계 결정 이유

   ## 알려진 제한사항
   - 솔직하게 미구현 부분이나 개선점 명시
   ```

3. **Git 커밋** - 구현 완료 후 변경사항을 커밋하세요:
   ```bash
   git add .
   git commit -m "구현: {approach_name}에 따른 {기능명}"
   ```

**지금 바로 구현을 시작하고, 위 3가지를 모두 완료하세요!**
