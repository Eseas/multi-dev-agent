# Multi-Agent Dev System - 시스템 개요

## 시스템 목적

여러 AI 에이전트가 협업하여 소프트웨어 개발 작업을 수행하는 자동화 시스템.
사람은 기획 단계에서 방향을 잡고, 이후 에이전트들이 구현·리뷰·테스트·통합을 자동 수행한다.

## 전체 흐름

```
[사람 + AI 기획] → [자동 구현 (N개 병렬)] → [자동 리뷰] → [자동 테스트] → [비교·선택] → [통합]
```

### Phase 0: 기획 (Human + AI)
- 사람과 AI가 대화하며 기획서 작성
- 구현 방향 N개를 결정 (1~3개, 적응형 또는 수동 지정)
- 완성된 기획서는 `planning/completed/`로 이동하여 실행 트리거

### Phase 1: 설계 (Architect Agent)
- 기획서를 분석하여 N개의 구현 접근법 상세 설계
- 각 접근법의 기술 스택, 구조, 핵심 로직 정의
- `approaches.json` 출력

### Phase 2: 구현 (Implementer Agents × N)
- N개의 에이전트가 각각 독립된 디렉토리에서 병렬 구현
- 각 에이전트는 할당된 접근법만 구현
- `work-done.md` + 실행 가능한 코드 출력

### Phase 3: 리뷰 (Reviewer Agent)
- 모든 구현체를 코드 리뷰
- 기획서 대비 완성도, 코드 품질, 잠재 이슈 평가
- `review-report.md` 출력

### Phase 4: 테스트 (Tester Agent)
- 각 구현체에 대해 테스트 실행
- 기능 테스트, 엣지 케이스, 성능 기본 측정
- `test-report.md` 출력

### Phase 5: 비교·선택 (Comparator Agent)
- 리뷰 + 테스트 결과를 종합 비교
- 최적 구현체 추천 (또는 하이브리드 제안)
- `comparison-report.md` 출력

### Phase 6: 통합 (Integrator Agent)
- 선택된 구현체를 root project에 통합
- 필요시 여러 구현체의 장점을 조합
- 통합 후 최종 테스트 수행

## 적응형 파이프라인

구현 개수 N은 작업 복잡도에 따라 자동 조정:

| 복잡도 | N | 기준 |
|--------|---|------|
| 단순 | 1 | 단일 해법이 명확, 버그 수정 등 |
| 보통 | 2 | 일반 기능 개발, 기본값 |
| 복잡 | 3 | 아키텍처 결정, 기술 스택 선택 |

기획서에서 명시적으로 N을 지정하면 해당 값 사용.
