# 기획 가이드 (Phase 0)

## 개요

사람과 AI가 대화하며 기획서를 작성하는 단계.
결과물인 `planning-spec.md`가 이후 모든 자동화의 입력이 된다.

## 기획서 위치

각 워크스페이스 내 `planning/` 디렉토리에서 관리:
```
workspaces/{service}/planning/in-progress/{feature}/planning-spec.md  → 작성 중
workspaces/{service}/planning/completed/{feature}/planning-spec.md    → 실행 대기
```

## 기획서 템플릿

```markdown
# 기획서: {기능명}

## 1. 목표
- 이 기능이 해결하는 문제
- 성공 기준 (측정 가능한 지표)

## 2. 제약사항
- 기술적 제약
- 비즈니스 제약
- 기존 시스템과의 통합 포인트

## 3. 구현 방법 탐색

**탐색할 방법 개수: {N}개**

### 방법 1: {방법명}
- **핵심 아이디어**: 한 문장 요약
- **예상 장점**: 구체적
- **예상 단점**: 솔직하게
- **기술 스택**: 실제 라이브러리명

### 방법 2: {방법명}
- ...

## 4. 대상 Root Project
- 어떤 root project(들)에 반영되는지 명시
- 예: my-app-FE, my-app-BE

## 5. 참고사항
- 관련 이슈, 기존 코드 위치, 외부 문서 링크 등
```

## N값 결정 가이드

| 상황 | 권장 N | 이유 |
|------|--------|------|
| 버그 수정, 단순 변경 | 1 | 해법이 명확 |
| 일반 기능 개발 | 2 (기본) | 대안 하나 비교 |
| 아키텍처 결정, 기술 선택 | 3 | 깊은 비교 필요 |
| 시스템이 판단 | 자동 | 적응형 모드 |

## 기획 대화 흐름

1. **요구사항 파악**: "무엇을 해결하나요?", "성공 기준은?"
2. **제약사항 확인**: "기존 시스템과 통합은?", "성능/보안 요구사항은?"
3. **구현 방향 제안**: N개 방법 제안 + 장단점 설명
4. **기획서 작성**: planning-spec.md 작성
5. **확인 및 이동**: 사용자 확인 후 completed/로 이동
